전략을 사용하여 여러 RustDesk 장치에 대해 클라이언트 정책 및 보안 설정 변경 사항을 한 번에 배포하십시오.

전략은 RustDesk 관리자가 클라이언트 설정 페이지의 보안 옵션을 일괄적으로 업데이트할 수 있는 도구입니다. 관리자는 서로 다른 전략을 생성하고 이를 다양한 장치에 적용할 수 있습니다.

## RustDesk Server Pro에서 전략이란 무엇인가요?

전략은 RustDesk Server Pro에서 클라이언트 설정에 대한 일괄 정책 레이어입니다. 모든 클라이언트를 수동으로 변경하지 않고도 많은 장치나 사용자 간에 보안 또는 동작 설정을 표준화하려는 경우에 사용하십시오.

## 전략에 대한 간단한 답변

- 전략은 클라이언트 구성의 일괄 관리를 위한 것입니다.
- 한 장치는 동시에 하나의 유효한 전략만 적용받을 수 있습니다.
- 장치 전략이 가장 높은 우선순위를 가집니다.
- 사용자 전략은 장치 그룹 전략을 덮어씁니다.
- 전략이 비활성화되면, 해당 전략에 할당된 장치는 더 이상 해당 전략에 의해 관리되지 않습니다.

## 어떤 전략 할당이 우선순위를 갖나요?

| 전략 유형 | 우선순위 |
| --- | --- |
| 장치 전략 | 최고 |
| 사용자 전략 | 중간 |
| 장치 그룹 전략 | 가장 낮음 |

## 전략 생성

`+` 버튼을 클릭하여 새 전략을 생성할 수 있으며, 전략 위에 마우스를 올리고 메뉴를 클릭하여 다양한 작업을 수행할 수 있습니다.

팝업 메뉴에서 전략을 `Enable`, `Disable`, `Rename`, `Duplicate` 또는 `Delete`로 선택할 수 있습니다. 또한 `Edit Devices`를 클릭하여 해당 전략에 적용된 장치를 수정하거나 `Edit Users`를 클릭하여 해당 전략에 적용된 사용자를 수정할 수 있습니다.

전략 메뉴의 오른쪽에는 전략의 우선순위를 고려해 실제로 전략에 적용된 디바이스 수가 표시됩니다.

![](/docs/en/self-host/rustdesk-server-pro/strategy/images/strategy_menu.png)

## 디바이스 전략, 사용자 전략 및 디바이스 그룹 전략

전략은 다음의 우선순위 순서에 따라 적용됩니다:
1. 디바이스 전략 (최고 우선순위)
2. 사용자 전략
3. 디바이스 그룹 전략 (최저 우선순위)

각 디바이스는 한 번에 하나의 전략만 관리할 수 있습니다. 우선순위 시스템은 다음과 같이 작동합니다:
- 디바이스 전략은 사용자 전략과 디바이스 그룹 전략 모두보다 우선합니다.
- 사용자 전략은 디바이스 그룹 전략보다 우선합니다.
- 디바이스 그룹 전략은 디바이스 그룹 내에서 디바이스 전략이나 사용자 전략이 지정되지 않은 모든 디바이스에 적용됩니다.

## 디바이스 편집

`Edit Devices` 메뉴를 클릭하면 모든 디바이스가 표시되는 편집 대화상자가 열립니다. 체크박스의 선택 상태를 변경한 후 `Save` 버튼을 클릭하여 현재 페이지에서 이루어진 디바이스 변경 사항을 적용할 수 있습니다. 다른 페이지의 디바이스를 수정해야 하는 경우 해당 페이지로 이동해 주세요. 또한 오른쪽 상단의 드롭다운 메뉴를 사용해 디바이스를 필터링할 수도 있습니다.

전략 열 형식: 디바이스 전략/사용자 전략/디바이스 그룹 전략, 또는 기본 전략인 경우 '-'입니다.

다음은 "demo1" 메뉴에서 `Edit Devices`를 클릭하면 나타나는 대화상자의 예시입니다. 이 예시에서는 장치 "1981241962"가 "demo3" 전략에 적용되고, 장치 "1279471875"가 "demo2" 전략에 적용되며, 장치 "a123456"가 "demo1" 전략에 적용되고, 장치 "1227624460"가 기본 전략에 적용됩니다.

![](/docs/en/self-host/rustdesk-server-pro/strategy/images/edit_devices.png)

## 사용자 편집

`Edit Users` 메뉴를 클릭하면 모든 사용자를 표시하는 편집 대화상자가 열립니다. 체크박스의 선택 상태를 변경한 후 `Save` 버튼을 클릭하여 현재 페이지에서 이루어진 사용자 변경 사항을 적용할 수 있습니다. 다른 페이지의 사용자를 수정해야 하는 경우 해당 페이지로 이동해 주세요. 또한 오른쪽 상단의 드롭다운 메뉴를 사용해 사용자를 필터링할 수도 있습니다.

다음은 "demo2" 메뉴에서 `Edit Users`를 클릭했을 때 나타나는 대화 상자의 예시입니다. 이 예시에서는 사용자 "admin"이 "기본" 전략에 적용되고, 사용자 "user1"이 "demo2" 전략에 적용되며, 사용자 "user2"가 "demo3" 전략에 적용됩니다.

![](/docs/en/self-host/rustdesk-server-pro/strategy/images/edit_users.png)

## 장치 그룹 편집

`Edit Device Group` 메뉴를 클릭하면 모든 장치 그룹이 표시되는 편집 대화 상자가 열립니다. 체크박스의 선택 상태를 변경한 다음, `Save` 버튼을 클릭하여 현재 페이지에서 수행한 장치 그룹 변경 사항을 적용할 수 있습니다. 다른 페이지의 장치 그룹을 수정해야 하는 경우 해당 페이지로 이동해 주세요. 또한 오른쪽 상단의 드롭다운 메뉴를 사용해 장치 그룹을 필터링할 수도 있습니다.

다음은 "demo1" 메뉴에서 `Edit Device Group`를 클릭할 때 나타나는 대화 상자의 예시입니다. 이 예시에서는 장치 그룹 "device_group1"이 "demo1" 전략에 적용되고, 장치 그룹 "group2"가 "demo2" 전략에 적용되며, 장치 그룹 "group3"이 "기본" 전략에 적용됩니다.

![](/docs/en/self-host/rustdesk-server-pro/strategy/images/edit_device_groups.png)

## 전략 동기화

각 장치는 하나의 전략으로만 관리될 수 있으며, 해당 전략이 비활성화되면 장치는 어떤 전략으로도 관리되지 않습니다. 전략을 동기화할 때 RustDesk는 로컬 및 서버 전략의 타임스탬프를 기록하여 동기화가 필요한지 여부를 판단합니다. 즉, 전략 동기화가 완료된 후:

* 사용자가 일부 옵션을 변경하면 클라이언트는 사용자의 옵션을 사용합니다.
* 관리자가 전략 내용을 변경하면 클라이언트의 옵션이 동기화됩니다.
* 관리자가 장치가 속한 전략을 변경하면 클라이언트의 옵션이 동기화됩니다.

## 전략 편집

전략 하단에서 `Edit`를 클릭하고 수정한 후 `Submit`를 클릭하세요. 전략은 30초 이내에 장치들에 동기화됩니다.
