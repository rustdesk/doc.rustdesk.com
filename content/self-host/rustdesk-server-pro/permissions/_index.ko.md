RustDesk Server Pro에서 어떤 사용자나 그룹이 어떤 장치에 연결할 수 있는지 결정하려면 접근 제어를 사용하세요.

## RustDesk Server Pro의 접근 제어란 무엇인가요?

접근 제어는 사용자나 그룹이 특정 장치에 접속할 수 있는지를 결정합니다. 이는 연결 시작 후 운영자가 할 수 있는 작업과는 관련이 없습니다. 세션 내 권한 설정을 위해서는 [Control Role](/docs/ko/self-host/rustdesk-server-pro/control-role/)를 사용하세요.

## 접근 제어에 대한 간단한 답변

- 접근 제어는 누가 어떤 장치에 연결할 수 있는지 결정합니다.
- 사용자 할당, 장치 그룹 및 그룹 간 접근 설정을 통해 작동합니다.
- 한 장치는 한 사용자, 한 장치 그룹 또는 둘 다에 할당될 수 있습니다.
- 비활성화된 사용자와 비활성화된 장치에는 접근할 수 없습니다.
- 장치 그룹 권한과 사용자 그룹 권한을 조합하여 접근을 허용할 수 있습니다.

## 가장 먼저 구성해야 할 사항은 무엇인가요?

1. 장치를 어떻게 할당할지 결정하세요: 직접 사용자에게 할당하거나, 장치 그룹에 할당하거나, 둘 다 할당할지 선택하세요.
2. 사용자가 자신의 그룹 외부의 장치에 접속해야 하는 경우 사용자 그룹 간 교차 규칙을 설정하세요.
3. 사용자별 할당보다 그룹 기반 장치 접근 관리가 더 쉬운 경우 장치 그룹 규칙을 추가하세요.

## 장치 접근 권한

장치는 단일 사용자, 단일 장치 그룹 또는 둘 다에 할당될 수 있습니다.

장치가 사용자에게 할당되면 해당 사용자, 사용자 그룹 또는 적절한 교차 사용자 그룹 설정을 통해 접근할 수 있습니다.

장치가 장치 그룹에 할당되면 적절한 교차 사용자-장치 그룹 설정을 통해 접근할 수 있습니다.

사용자에게 장치를 할당하는 방법은 세 가지가 있습니다:
- 콘솔 장치 페이지를 통해
- 클라이언트 측에서 지정된 사용자 계정에 로그인하여
- 명령줄을 통해 할당

장치를 장치 그룹에 할당하는 방법은 두 가지가 있습니다:
- 콘솔 장치 페이지를 통해
- 명령줄을 통해 할당

다음 두 가지 상황에서는 장치에 접근할 수 없게 됩니다:
- 콘솔 장치 페이지에서 장치 `disable`로 설정
- 콘솔 사용자 페이지에서 사용자 `disable`로 설정

## 사용자 그룹 접근 설정

웹 콘솔의 그룹 페이지로 이동하여, 아래와 같이 `Edit`를 클릭해 교차 그룹 설정을 편집하십시오.

`Access with other groups`에 대한 귀하의 수정 사항은 `OK` 버튼을 클릭할 필요 없이 즉시 적용됩니다.

`Can access to`와 `Can be accessed from`는 거의 동일한 기능을 수행하며, 편의를 위해 두 옵션을 모두 제공합니다. 그러나 이로 인해 약간의 혼란이 발생할 수 있습니다.

## 디바이스 그룹 액세스 설정

디바이스 그룹은 액세스 권한을 관리하는 또 다른 방법을 제공합니다. 주요 규칙은 다음과 같습니다:

1. 하나의 디바이스는 오직 하나의 디바이스 그룹에만 추가될 수 있습니다.
2. 사용자 또는 사용자 그룹에 대해 디바이스 그룹의 액세스 권한을 설정할 수 있습니다. 이러한 권한은 사용자 그룹의 액세스 권한과 누적됩니다—즉, 사용자 그룹 권한이나 디바이스 그룹 권한 둘 중 하나라도 허용한다면 액세스가 부여됩니다.
3. 할당되지 않은 디바이스를 디바이스 그룹에 추가하면 더 이상 "할당되지 않음" 상태로 간주되지 않습니다.

{{% notice note %}}
디바이스 그룹 기능을 사용하려면 RustDesk 클라이언트 버전 1.3.8 이상 및 RustDesk 서버 프로 버전 1.5.0 이상이 필요합니다.
{{% /notice %}}
